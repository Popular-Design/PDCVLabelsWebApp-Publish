# Security Policy

## Reporting a Vulnerability

The security of PDCVLabelsWebApp is important to us. If you discover a security vulnerability, please follow these steps:

### How to Report

1. **Do not** open a public issue for security vulnerabilities
2. Contact the maintainers directly through:
   - GitHub Security Advisories (preferred)
   - Direct message to repository maintainers

### What to Include

When reporting a vulnerability, please include:
- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested fixes (if available)

### Response Timeline

- We will acknowledge receipt of your vulnerability report within 48 hours
- We will provide an initial assessment within 5 business days
- We will work on a fix and keep you updated on progress
- Once fixed, we will publicly disclose the vulnerability (with credit to you, if desired)

## Supported Versions

As this is a publishing repository, security updates will be applied to the latest published version from the private source repository.

## Security Best Practices

When using PDCVLabelsWebApp:
- Always use the latest published version
- Follow secure configuration practices outlined in documentation
- Keep dependencies up to date
- Report any security concerns promptly

## Disclosure Policy

We believe in responsible disclosure and will:
- Work with security researchers to understand and fix vulnerabilities
- Provide credit to researchers who report valid security issues
- Publish security advisories for significant vulnerabilities

Thank you for helping keep PDCVLabelsWebApp secure!
